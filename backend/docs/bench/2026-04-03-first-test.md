## 现象与结论
- 现象：出现大量 code 0，主要因为服务端响应时间超时（timeout）。
- 现象：Lua 成功后同步写 MySQL（插订单）变慢，导致请求排队，RT 拉长。

## 一致性观察
**Redis**

```text
GET coupon:stock:9 = 8234
```

**MySQL**

- 落库订单数：827

**结论**
- Lua 扣减和 DB 落库仍然不同步，存在少卖/丢单现象。
- 推测原因：DB 读写慢、请求超时导致客户端认为失败，但服务端后台仍在写入。

## 下一步改进方向
- 下单落库改为异步：HTTP 层只做校验 + Redis 原子扣减 + 记录成功事件，由 worker 写 MySQL。
- 增加对账/补偿：定期对比 Redis 扣减数与 MySQL 订单数，做补写或回补库存。
- 优化 MySQL：索引、连接池参数、写入路径（必要时批量写/队列化写）。
