## 现象与结论
- **性能飞跃**: 迁移至 Linux 并调优 `somaxconn` 等内核参数后，系统彻底解决了 Windows 环境下的连接阻塞问题。
- **万级 QPS 达成**: 单机环境下实现 10000 QPS 稳定吞吐，且 P99 保持在 150ms 以内，表现出极佳的响应稳定性。
- **架构优势**: 证明了 **Redis Lua 脚本** 进行库存扣减与 **Redis Stream 异步批量落库** 架构在超高并发下的卓越性能。

## 一致性观察
- **Redis**: `coupon:stock:1` 准确减少 50,000，剩余 50,000。
- **MySQL**: `seckill_orders` 订单数最终精准达到 50,000，异步落库无遗漏。
- **死锁处理**: 在 10000 RPS 极端压力下，Worker 虽偶发 Deadlock 报错，但通过 Redis Stream 的重试机制实现了数据的最终一致性。
